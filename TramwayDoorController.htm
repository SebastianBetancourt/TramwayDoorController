Project report
Project for the Vérification de modèles course
A model for the tram door controller



Summary:
This is a report about the final project of the Vérification de modèles course offered at Université de Namur with the advice of James Ortiz as professor of the course and Jesús Aranda at Universidad del Valle as professor who guides an investigative practice.
This document presents a model for a tram door controller with bridge and buttons for operation. The model is made by using state machines with the Scade tool. Moreover, the model has verifications of its correct behavior using the model checker available in Scade, where we verify the success of two relevant properties.
Authors: Jaime Cuartas Granada and Joan Sebastian Betancourt Arias
Date: April 2020

Controller
Table 15: States of Controller_1
State Comments and Information
Controller:Bridge Use the Bridge state machine to perform retraction and deployment actions
Controller:BridgeButton Use a button state machine to simulate its press that will deploy the bridge. Pressing the button will request the door to be closed regardless of its state, and once the bridge is deployed in the Bridge state, the door will be automatically requested. In this way, the request for the bridge will imply the request for the door later.
Controller:Door Use the Door state machine to perform closing and opening actions
Controller:DoorButton Use a button state machine to simulate its press that will open the door. This state implies that eventually the state of the door will be opened, however it does not imply that the bridge
Controller:Open Represents the possibility of passengers entering to the tram. In this state waits for immDeparture to leave the station or the bridgeRequest in case it has not been deployed yet.
Controller:ToLeaveStation Once it is secured to leave we only wait for the tram to be at another station again to start again.
Controller:ToPressButton It is the initial state. This state waits for the door or bridge request.

3.2.1.1. Interface
Table 22: Inputs of SafeBridge
Name Type Comments and Information
requireDoor bool Controller input
requireBridge bool Controller input
immDeparture bool Controller input
inStation bool Controller input
Table 23: Outputs of SafeBridge
Name Type Comments and Information
Output bool Represents:
(bridgeStatus.closing ∨ bridgeStatus.opening) → (¬(doorStatus.opened))
The door can not be opening if the bridge have an action. That means that any action, such as closing or opening, implies that the door is not opened.
